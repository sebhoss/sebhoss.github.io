Formatting rules for SQL statements.
Show bad examples. Introduce a lot of whitespace, show smaller, consistent layout? Consistent is good because of pattern matching.

h2. General Rules

# No implicit joins!
# Columns are aligned

h2. SELECT-Statements

{% highlight sql %}
SELECT column1,
       column2,
       column3
FROM table1
     INNER JOIN table2
       USING (column1, column2)
     INNER JOIN table3
       ON  table3.column3 = table1.column4
       AND table3.column5 = table2.column6
WHERE column5 < 0
{% endhighlight %}

h2. INSERT-Statements

{% highlight sql %}
INSERT INTO table
    (column1, column2, column3)
VALUES
    (1, 2, 3)
{% endhighlight %}

h2. UPDATE-Statements

{% highlight sql %}
UPDATE table
SET column1 = 1,
    column2 = 2,
    column3 = 3
WHERE column4 > 0
{% endhighlight %}
